# AI 功能说明

AI 功能包括供应商配置、AI 优化和 AI 问答。它们围绕当前信笺工作，生成记录会随 `.letterpaper` 文件保存。

## 供应商与模型

Gemini、DeepSeek 和 Codex CLI 是内置供应商。用户还可以添加多个相互独立的自定义供应商：

- OpenAI 兼容：使用 Chat Completions 协议，Base URL 不应包含 `/chat/completions`。
- Anthropic 原生：使用 Messages API，Base URL 不应包含 `/messages`。

配置流程：

1. 从顶部 AI 入口打开 AI 设置。
2. 选择内置供应商，或点击“添加供应商”填写唯一名称、协议和 Base URL。
3. 在详情页填写 API Key，并添加模型名称和模型标识。
4. 逐个测试模型连接。
5. 将测试成功的模型设为默认。

没有已测试模型的供应商会显示未配置，不能设为默认或用于生成。内置供应商不可删除；自定义供应商可以删除，但默认项必须先切换。删除供应商会同时移除本机保存的密钥和模型配置。公开配置不会返回完整 API Key。

OpenAI 兼容请求使用 Bearer 鉴权并解析 Chat Completions SSE；Anthropic 请求使用 `x-api-key` 和 `anthropic-version: 2023-06-01`，将 system 消息提升为顶层字段，并把用量映射为现有的输入、输出和总 token 统计。

## Codex CLI

Codex CLI 使用本机已经安装并登录的 Codex，不需要在笺间填写 Base URL 或 API Key。

1. 安装：`npm install -g @openai/codex`。
2. 在 Codex CLI 配置页点击“重新检查”。
3. 未登录时点击“登录 Codex”，在打开的终端完成授权。
4. 再次检查后，笺间会同步当前账号可用的模型和每个模型支持的推理强度。
5. 为各模型选择默认推理强度，再设置默认模型或默认供应商。

Codex CLI 仅在桌面端可真实调用。笺间不保存 Codex 登录凭据；模型、说明和推理强度以本机 CLI 返回的目录为准。刷新暂时失败但存在上次成功目录时仍可继续使用，并显示过期提示。

生成时每次创建独立临时会话，Prompt 通过标准输入传递，保留 `--sandbox read-only`、禁止审批、忽略用户规则与配置。Codex 可以读取本次允许范围内的项目文件，但不能修改文件；调用消耗当前登录账号的额度。

## AI 优化

AI 优化用于润色当前信笺、调整结构与表达：

- 定稿线以上作为已定稿背景，线以下是本次优化重点。
- 优化记录按信笺保存，退出 AI 模式或切换信笺不会清除。
- 开始新的优化会覆盖当前信笺已有结果。
- 清空优化只影响当前信笺，不会清除 AI 问答。

### 直接应用

优化模型只负责生成自由 Markdown，不承担原文位置标注。“直接应用定位模型”使用当前正文块清单、选中的优化块和相邻优化块判断替换或插入位置；正文修改始终由本地编辑器完成。

- Gemini、DeepSeek 定位请求固定使用 JSON 输出，并分别使用模型允许的最大输出上限；这个上限只代表最多可输出多少，不会要求模型实际输出到该长度。Codex CLI 通过提示词约束输出。
- 返回格式或位置字段错误时会自动修正一次；无法可靠定位、低可信度或网络失败时切换为“选择位置应用”。
- 手动模式可点选可编辑原文块，再选择替换、插入到前面或插入到后面；定稿区和保护块不可选，Esc 可取消。
- 定位成功后不会立即修改正文：待替换原文以红色显示，拟应用内容以蓝色显示，用户可直接在正文内确认或取消；插入操作只显示蓝色拟插入内容。修改方式、目标摘要和评注影响会一并显示。
- 只有在正文内确认时才产生一个编辑事务，可用一次 Ctrl+Z 完整撤销；取消对比不修改正文。定位期间若仅修改了无关段落，会尝试按目标原始内容和相邻上下文重新定位；目标本身变化或出现歧义时不会自动写入。
- 定位请求不包含工作区、研究资料、其他信笺、文件路径或完整 AI 历史。

## AI 问答

AI 问答用于围绕当前信笺追问、审阅、找漏洞、生成标题和局部改写：

- 消息记录、输入草稿和已标记文字按信笺保存。
- 切换模型后保留当前信笺记录；切换信笺时显示该信笺自己的 AI 状态。
- 清空只移除当前信笺的问答，不影响 AI 优化、Codex 目录范围或图片模式。

### Codex 目录范围

当前模型为 Codex CLI 时，“切换模型”右侧显示目录范围：

- 仅当前信笺：为本次请求使用空临时目录，只发送编辑器内存中的当前正文和已标记文字。
- 信笺所在目录：使用已保存信笺的父目录；未保存信笺不可选。
- 整个工作区：使用当前工作区根目录，兼容旧信笺的默认行为。
- 选择工作区子目录：只保存相对工作区路径，并通过目录树选择。

主进程会校验真实路径、目录存在性和工作区包含关系，拒绝 `..`、绝对路径、失效目录及逃出工作区的符号链接。显式范围失效时会提示重新选择，不会自动扩大到整个工作区。目录范围按信笺保存，只影响下一次提问。

### 当前信笺原图

目录范围菜单底部提供“信笺图片”：

- 原图：默认模式，保留正文中的 `[图N.标题]` 锚点，并按图号顺序通过 Codex CLI `--image` 附加全部原图。
- 仅标题：只发送图号和标题，不附加图片文件。

原图来自当前编辑器内存，支持未保存的 data URL 和已保存信笺的内嵌资源，不要求 Codex 自行读取或解包 `.letterpaper`。任一图片无法读取或格式无效时，请求会指出具体图号并停止；不会静默漏图或自动降级。临时图片在完成、失败或取消后都会清理。

HTTP 供应商仍只接收图片标题；原图模式只作用于 Codex AI 问答，不改变 AI 优化。

## 保存与运行时回退

- AI 状态随 `.letterpaper` 保存，复制信笺时会一起移动。
- AI 状态变化会让信笺进入未保存状态，需要保存或等待自动保存。
- 历史信笺引用的供应商或模型已不存在时，运行时回退到当前默认可用模型，不改写历史记录。
